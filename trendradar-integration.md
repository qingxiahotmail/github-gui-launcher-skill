# GitHub GUI 启动器技能 - TrendRadar 集成示例

## 集成概述

本文档介绍如何在 TrendRadar 项目中集成和使用 GitHub GUI 启动器技能，作为技能的实际应用案例。

**重要说明**：GitHub GUI 启动器技能是一个独立的工具，可以在任何需要的场景中使用，不限于 TrendRadar 项目。

## 集成目标

- 展示 GitHub GUI 启动器技能在实际项目中的使用效果
- 验证技能对小白用户的友好程度
- 提供一个完整的技能集成参考案例

## 集成步骤

### 1. 获取 TrendRadar 项目

使用 GitHub GUI 启动器技能打开 TrendRadar 项目：

```bash
# 通过技能打开 TrendRadar 仓库
python skills/github_gui_launcher/skill.py --github_path "https://github.com/sansan0/TrendRadar"
```

### 2. 安装项目依赖

按照 TrendRadar 项目 README 中的说明安装必要的依赖：

```bash
# 安装 Python 依赖
pip install -r requirements.txt

# 或使用 uv 包管理器
uv sync
```

### 3. 配置项目

根据你的需求修改配置文件：

```bash
# 编辑配置文件
config/config.yaml
```

### 4. 运行项目

使用技能启动项目，开始监控热点新闻：

```bash
# 运行项目
start_trendradar_simple.bat
```

## 常见问题

### 客户端未找到

**症状**：技能执行失败，提示 "GitHub GUI 客户端未找到"

**解决方案**：
1. 确认已安装 GitHub Desktop 或其他支持的客户端
2. 检查客户端安装路径是否正确
3. 在 `config.json` 中手动配置客户端路径

### 路径不存在

**症状**：技能执行失败，提示 "路径不存在"

**解决方案**：
1. 确认输入的本地路径或 GitHub URL 正确
2. 检查本地路径是否存在
3. 检查 GitHub URL 是否可访问

### 权限不足

**症状**：技能执行失败，提示 "权限不足"

**解决方案**：
1. 以管理员身份运行命令
2. 检查客户端执行权限
3. 确保用户有足够的权限访问指定路径

## 注意事项

1. **技能独立性**：GitHub GUI 启动器技能可以独立使用，无需依赖 TrendRadar
2. **客户端要求**：确保已安装至少一种 GitHub GUI 客户端
3. **路径配置**：对于自定义客户端路径，请确保路径正确且具有执行权限
4. **网络连接**：确保网络连接正常，以便访问 GitHub 仓库
